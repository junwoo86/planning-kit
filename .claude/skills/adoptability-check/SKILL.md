# adoptability-check — 채택(adoptability) 선검증 (S10)

방법론 SSoT = [guideline](../../../docs/planning-harness-guideline.md) §13. 절차만 실행한다.

## 산출물
- `09-readiness.md` — `readiness: PASS|FAIL` 표식 필수(헤더 `<!-- stage: S10 | status: done -->`).

## 절차 — '현업 준비도(readiness)' 검증
1. **즉시 이해**: 실무자가 **설명 없이** 이해하는가 — *측정*: 첫 사용자 n≥5명이 도움 없이 핵심 과제를 30초 내 완료, 성공률 ≥80%(미달 = 미완). Nielsen 휴리스틱·Krug "자명함"을 잣대로.
2. **워크플로 적합**: 기존 워크플로(S5 as-is)에 자연스럽게 끼는가. **별도 행동을 강요하면 = FAIL**.
3. **운영·교육·롤아웃**: 누가 굴리고 누가 가르치나. **단계적 롤아웃 계획이 미정이면 = FAIL**(빅뱅 금지).
4. **판정(AND)**: `readiness: PASS` = 절차1(n≥5·30초·성공률≥80%) **AND** 절차2(별도행동 강요 없음) **AND** 절차3(롤아웃 계획 존재) 전부 충족 시에만. 하나라도 미달 = **'미완'**. 목업 전(실측 불가)이면 PASS 금지 → **`readiness: 잠정`** + "Figma 후 실측" 표시. 임계값(n·초·%) 기본값 조정 시 근거를 09-readiness에 기록. 템플릿 = `templates/readiness.md`.

## 충실도 주의 (Figma 분업)
- 30초 이해 측정은 *실제 화면*을 봐야 가능 → **목업/Figma 와이어프레임이 있어야 완전 검증**.
- 목업 전(보고서 단계)에는 워크플로/엣지 기반 **잠정 readiness**만 기록하고 "Figma 후 보완" 표시. 이 경우 보고서 게이트는 cond5를 N/A로 둔다.

## 산출 규약
- 측정 결과(n·성공률·소요)와 미달 항목을 명시. 모든 줄 출처 태그.
- `_state.md`: S10 상태 갱신. PASS면 S11/S12 완성 경로, 미달이면 '조건부'.
