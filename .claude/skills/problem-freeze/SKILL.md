# problem-freeze — 문제 정의·동결 + 성공기준 (S2–S3)

방법론 SSoT = [guideline](../../../docs/planning-harness-guideline.md) §5(문제 동결)·§6(성공기준)·§1(4대 원칙). 절차만 실행한다.

## 산출물
- `01-problem-set.md` — **동결** 문제집합(헤더 status: frozen).
- `02-success.md` — 계층형 3층 성공기준(status: done).

## 절차 — S2 문제 정의 & 동결
1. **증상을 그대로 받지 않는다.** "X 자동화" → **5 whys**로 근본원인까지(guideline §5). **멈춤 기준(actionable-root, T2)**: ① 통제 밖 도달 OR ② 원인이 반복됨 OR ③ 그것을 해소하면 증상이 재발하지 않음 — 중 *먼저 닿는 곳*에서 멈춘다. 분기하면 *동결 문제집합 영향이 큰 가지*를 우선. 근본원인 진술로 문제를 재서술.
2. **양방향 다리 — 전략 승격(여기 주입)**: 실무발 입력이면 동결 *전* 강제 질문 — "이 작업이 어떤 더 큰 문제/프로세스 파괴로 이어지나?" 근본원인이 한 층 위일 수 있다(§5).
3. **현업 검증**: 추출 문제를 녹취/관찰·인터뷰로 *실제 그 문제가 맞는지* 대조. **자료/관찰/인터뷰 부재 시(T3, 현업자료 부재 동결경로)**: 진행을 막지 않되 **잠정 동결(provisional)** — 해당 문제에 `[현업검증=확인불가]` 태그 + S8 검증필요가정 등재 + 종료게이트 '완성' 불가('조건부' 강제). 조용한 폴백 금지.
4. **문제마다 기록**: 진술(근본원인 수준)·영향 주체·현재 상태(왜 부족)·*안 풀면 드는 비용*.
5. **동결**: 문제집합을 명시 확정. 헤더 status를 `frozen`으로. 이후 추가·변경은 **변경요청 프로토콜**(ASK→재동결→S3·S6 하류 재실행→provenance 기록)으로만(§5). 조용한 범위 이동 금지.
   > 이 동결 집합 = S6 정합성 매트릭스의 **행**. 라벨을 그대로 매트릭스에 쓴다.

## 절차 — S3 성공기준 (guideline §6)
계층형 3층 각각에 측정가능 지표:
- **③ 근본원인 해소**: 증상 아닌 *원인 지표*. 구분법 — "이 지표가 좋아져도 근본원인이 그대로면 재발하는가?" 재발하면 증상 지표.
- **② 핵심지표**: North Star + 입력지표 3~5 + **사전약속 임계값**("지표 X가 기한 Y까지 Z 안 움직이면 실패")을 *출시 전* 선언. 사후 목표선 이동 금지. 베이스라인 없으면 측정을 선행 과제로 두거나 proxy를 `추정` 태그로.
- **① 현업 채택**: 실무자가 실제로 쓰는가(활성·재방문·대체 폐기율).
- **모호표현 금지**: "실시간·100%·빠르게·직관적" 등은 *측정 불가* → `→측정:` 또는 `[정의됨]`으로 SMART 재정의해야 종료 게이트(completion-gate cond6)를 통과한다.
- **non-goal 명시** + 선행/후행지표 분리.
