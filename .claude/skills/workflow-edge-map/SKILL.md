# workflow-edge-map — 워크플로 & 엣지케이스 포착 (S5)

방법론 SSoT = [guideline](../../../docs/planning-harness-guideline.md) §8. 절차만 실행한다.

## 산출물
- **입력**: `00-intake.md`(입력유형)·`01-problem-set.md`(동결 문제). **산출**: `04-workflow-edge.md`(헤더 `<!-- stage: S5 | status: done -->`). **하류 소비처**: S6 coherence-matrix·S10 adoptability(as-is 적합)·S8 보고서.

## 절차
1. **양방향 다리 — 현업 검증 방향(여기 주입)**: 비전발 입력이면 as-is·엣지·채택가능성으로 *현업 검증*을 강제한다(대표가 못 보는 나무).
2. **as-is 매핑(실제 실행)**: 이론 절차가 아니라 *현재 진짜 일어나는* 흐름 — 비공식 우회·스프레드시트 편법·반복 확인까지. 자료 부족 시 관찰/인터뷰. **발굴 멈춤 기준(actionable-root)**: 우회·편법을 *통제 밖에 닿거나 패턴이 반복될 때까지* 내려가되 그 지점에서 멈춘다. **현업 자료/관찰 부재 시(T3)**: 잠정 매핑 + `[현업검증=확인불가]` 태그 + S8 검증필요가정 등재(조용한 폴백 금지).
3. **to-be 매핑**: 개선된 미래 흐름 + 전환(to-do) 액션.
4. **엣지케이스 강제 질문(각 단계마다)**: *뭐가 잘못되나 / 오프라인 / 권한 만료 / 중단 후 재개 / 부분 실패 / 동시 실행 / 빈 입력*. 에러·예외 누락을 구조적으로 차단.
5. **산출**: 각 화면/영역의 **상태**(초기·로딩·빈·성공·에러·권한없음) + **예외 목록**. → feature-spec(fs§5·fs§8) 매핑 대비.
