# reference-scout — 레퍼런스 인지 주입 (S4) ★

방법론 SSoT = [guideline](../../../docs/planning-harness-guideline.md) §7. 절차만 실행한다.

## 산출물
- **입력**: `01-problem-set.md`(동결 문제집합 — S2 이후에만). **산출**: `03-references.md`(헤더 `<!-- stage: S4 | status: done -->`). **하류 소비처**: S6 coherence-matrix(채택 후보 입력)·S8 보고서 §5.

## 절차
1. **타이밍 게이트**: 문제가 정의되기 *전*이면 노이즈 → **S2 동결 이후에만** 발화한다.
2. **무엇을 찾나**: 같은/유사 문제를 푼 제품·시스템(업계 안팎) · **교차 도메인 유추**(다른 산업의 유사 해법) · 경쟁/선례 **teardown**(핵심 기능·설계 결정·*왜*).
3. **'실사용 검증' 필터 + 증거 기준(엄격 카운팅, T11)**: vaporware·마케팅 아님. **채택 신호 = 제3자 독립 주체(실사용 고객·독립 리뷰·도입 사례)만 1로 카운트**. 벤더 자기발표·보도자료·재인용은 **합산해서 1**, 단종·베이퍼웨어·미출시는 **제외**, 출처/독립성이 모호하면 **한 단계 강등**. → **독립 신호 2+ = `확실` / 1 = `추정` / 0 = `확인불가`**(확인불가 위에 권고 금지). 탐색 = `WebSearch`/`WebFetch`(허용 도메인) → 깊으면 **reference-scout 에이전트** 또는 비-devkit `deep-research` 위임.
4. **unknown-unknowns 드러내기**: "당사자도 *모른다는 걸 모르는*" 부분을 의도적으로 묻는다(Rumsfeld unknown-unknown).
5. **카고컬트 차단(필수 게이트)**: "저기서 먹혔으니 우리도"는 **금지**. 모든 레퍼런스 통찰은 §9 정합성 매트릭스 + 동결 문제집합 + 참조 컨텍스트 맵(§10)을 **통과해야** 채택 후보다. 레퍼런스는 인지를 넓히되 게이트를 우회하지 못한다.
6. **레퍼런스 부재 처리(선례 0, T11)**: 선례 없으면 진행은 막지 않되 — ① 핵심 해법 가정을 **Unknown+Important로 S8에 강제 등재** ② **근거 충분성 점수 상한 6/10**(스코어카드) ③ '선례 없음 = 불확실성↑' 플래그. *검증된 듯* 진행 금지.

## 산출 규약
- 각 레퍼런스: 무엇을·핵심 설계결정·*왜*·채택 신호 수·확신도 태그([확실]/[추정]/[확인불가]). 모든 줄 출처 태그.
- `_state.md`: S4 상태 갱신. 채택 후보는 S6 매트릭스 입력으로 표시.
