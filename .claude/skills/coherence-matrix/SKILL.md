# coherence-matrix — 해법 설계 + 문제 정합성 매트릭스 (S6)

방법론 SSoT = [guideline](../../../docs/planning-harness-guideline.md) §9. 템플릿 = [.claude/templates/coherence-matrix.md](../../templates/coherence-matrix.md). 절차만 실행한다.

## 산출물
- `05-coherence-matrix.md` — 살아있는 정합성 원장(템플릿 복사 후 채움).

## 절차
1. **매트릭스 구성**: 행=`01-problem-set.md`의 동결 문제(라벨 동일), 열=기획 요소. 각 칸 = `+`(개선)/`0`(무영향)/`-[id]`(악화). 요소를 추가·변경할 때마다 갱신.
2. **악화(−) 칸 규칙 — "대안 먼저, 안 되면 하드게이트"**(§9):
   1. 악화 감지 → 먼저 **악화 없는 대안 1~3개** 제시(2차 효과로 숨은 악화도 "그다음 뭐가?"로 재귀 발굴; TRIZ 모순 매트릭스 활용). **재귀 멈춤 기준(actionable-root)**: 최소 **2차 효과까지 필수** — 통제 밖 도달 OR 원인 반복 OR 해소 시 재발 안 함 중 먼저 닿는 곳.
   2. 대안으로 해소되면 채택. 안 되면 **완화책**(노브·탈출구·후속작업)을 원장에 *구조화 필드*로 명시 — 담당·구체행동·결정기록. 플레이스홀더("추후"·"미정") 금지.
   3. 완화책도 없으면 status `blocking`으로 두고 **사람 결정을 강제 + `_state.md` 결정 원장에 기록**. 조용히 넘어가지 않는다.
3. **상호배타 문제**(요소가 아니라 *문제끼리* 충돌, 어떤 요소로도 둘 다 +가 불가): 완화책 대상 아님 → **사람에게 재범위화(re-scoping) 강제**(한 문제 우선순위 하향/분리 + 기록). status `rescoped`.
4. **우선순위(다수일 때)**: 경량 RICE/ICE로 정량 정렬하되 *가중치를 평가 전에 고정*(역설계 방지). 전사 파급(§10)과 결합.
5. **카고컬트 게이트 연계**(§7): S4 레퍼런스에서 온 통찰도 이 매트릭스 + 동결 문제집합 + 참조맵을 통과해야 채택 후보. "저기서 먹혔으니"는 게이트를 우회 못 한다.

## 결정적 검사 (반드시 실행)
```
python3 .claude/tools/coherence-check.py docs/planning/<slug>
```
- 모든 `-[id]` 칸이 구조화된 원장 항목을 갖는지, 맨 `-`(완화책 없는 악화)가 없는지, 플레이스홀더가 없는지 *결정적* 판정.
- exit 0 = PASS. FAIL이면 해소 전엔 다음 단계로 가지 않는다.
- 완화책의 *적절성*·근본원인 *해소 여부* 같은 의미 판단은 이 도구가 하지 않는다 → 고위험(T2)이면 독립검토(coherence-auditor, P2)로.
