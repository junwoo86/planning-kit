# inputs/ — 기획/보고서를 *만들* 대상의 원본

여기는 **킷으로 기획·보고서를 생성할 대상**의 원본 자료 자리입니다. 킷이 읽어 처리하고, 산출물은 `docs/`로 갑니다(섞지 않습니다).

## 어디에 둘지 — 한 질문으로 갈립니다
**"이 자료를 킷으로 *기획/보고서 생성*할 건가?"**
- **예 (작업 대상)** → 여기 **`inputs/<주제>/`**. 킷이 읽어 처리하고 참조 맵에 등재합니다.
- **아니, 그냥 *참조*만** (이미 확정됐거나 다른 데서 진행 중) → `docs/reference-context-map/`에 **직접** 둡니다
  (확정=`finalized/`, 진행중·의견=`open/`; 맨 위에 `<!-- reference-context: <도메인> | status: 확정|검토|컨셉 | source: 참조 | as-of: <날짜> -->` 한 줄).

## 전략기획 맥락 (S0 입력)
기획을 시작할 때, 가진 자료를 **`inputs/<주제-slug>/`** 하위 폴더에 넣으세요.
- slug = 영문 kebab-case 주제. 예: 견적 자동화 → `inputs/quote-automation/`
- 한 덩어리면 `inputs/<slug>/context.md` 한 파일로 충분합니다. 녹취·기존 기획서·메모 여러 개여도 됩니다.
- 그런 뒤 채팅에 `/strategy-plan` — 킷이 이 폴더를 전부 읽고 시작합니다.
- 자료가 없어도 시작됩니다(큰 목표만 있어도) — 인터뷰로 끌어냅니다.

자세한 사용법은 루트 [README.md](../README.md) 참고.

## 기타 원본 (선택)
- 특정 기획 건과 무관한 **프로젝트 레벨 참조 원본**이 있으면 여기 둘 수 있습니다(예: 기술 검토 가이드라인, 별도 행동지침 등 — *예시이며 현재 저장소엔 없음*).
